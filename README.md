# MLflow Starter Kit 🚀

Welcome to the **MLflow Starter Kit**! This repository is designed to help you quickly understand and implement [MLflow](https://mlflow.org/), an open-source platform for managing the end-to-end machine learning lifecycle.

## 🎯 What is MLflow?

MLflow is a tool that helps machine learning engineers:

- **Track Experiments**: Log parameters, metrics, and models from training runs
- **Manage Models**: Store, version, and register trained models
- **Deploy Models**: Package models for production use
- **Collaborate**: Share results and models with your team

This project includes beginner-friendly, well-commented Jupyter Notebooks that demonstrate how to:

- Connect to an MLflow Tracking Server.
- Create experiments and log multiple runs.
- Train machine learning models (Logistic Regression, Random Forest, Neural Networks).
- Track model parameters, metrics, and models using MLflow.

## 📁 Repository Structure

### Root Level Files:

- **`requirements.txt`**: Python dependencies and libraries needed to run this project
  - mlflow: The main MLflow library for experiment tracking
  - scikit-learn: Machine learning toolkit for training models
  - tensorflow/keras: Deep learning library for neural networks
  - pandas/numpy: Data manipulation libraries
  - hyperopt: Hyperparameter optimization library

- **`README.md`**: This file! Documentation for the project

- **`get-started.ipynb`**: **START HERE!**
  - A simple, beginner-friendly introduction to MLflow
  - Shows how to connect to the MLflow Tracking Server
  - Demonstrates how to log basic metrics and track runs
  - Great for understanding MLflow fundamentals (5-10 minutes)

### Folder: `1-MLproject/`

This folder contains practical ML projects with increasing complexity:

- **`gettingstarted.ipynb`**: Training a Logistic Regression Model on Iris Dataset
  - Loads the classic Iris flower dataset
  - Trains two different Logistic Regression models with different hyperparameters
  - Shows how to track parameters, metrics, and model artifacts in MLflow
  - Compares multiple runs to find the best model
  - Good for learning MLflow workflow (15-20 minutes)

- **`housepricepredict.ipynb`**: House Price Prediction with Hyperparameter Tuning
  - Uses the California Housing dataset (real-world data!)
  - Demonstrates GridSearchCV for hyperparameter tuning
  - Shows how to test multiple model configurations automatically
  - Logs all experiments to MLflow for easy comparison
  - Teaches advanced MLflow features (30-45 minutes)

- **`mlflow.db`**: Local SQLite database storing experiment metadata
  - This file automatically creates experiment records
  - Can be deleted safely (will be recreated)

- **`mlruns/`**: Directory containing all logged model runs and artifacts
  - Auto-generated by MLflow
  - Contains model files, metrics, parameters for each run

### Folder: `2-DLFLOW/`

Advanced deep learning with MLflow:

- **`starter.ipynb`**: Wine Quality Prediction using Deep Learning & Hyperparameter Optimization
  - Loads wine quality dataset from the web
  - Builds an Artificial Neural Network (ANN) using Keras/TensorFlow
  - Uses HyperOpt for intelligent hyperparameter search (not just grid search!)
  - Shows nested MLflow runs for tracking multiple configurations
  - Deploys the best model to a REST API
  - Most advanced notebook in the project (45-60 minutes)

- **`mlruns/`**: Contains records of deep learning experiments
  - Multiple experiment runs tracked with different hyperparameters
  - Model artifacts and performance metrics

## 🚀 Setup Instructions

Follow these steps carefully to get the project running on your local machine:

### 1. Clone or Download the Repository

**Using Git (recommended):**

```bash
git clone https://github.com/YOUR_USERNAME/mlflow-starter-kit.git
cd mlflow-starter-kit
```

**OR if downloading as ZIP:**

- Download the ZIP file and extract it
- Open your terminal and navigate to the extracted folder

### 2. Create and Activate a Virtual Environment

A virtual environment keeps this project's dependencies isolated from others on your computer.

**On Windows (PowerShell):**

```bash
python -m venv .venv
.\.venv\Scripts\Activate.ps1
```

**On macOS/Linux (Terminal):**

```bash
python3 -m venv .venv
source .venv/bin/activate
```

✅ Check: You should see `(.venv)` at the beginning of your terminal line after activation.

### 3. Install All Required Dependencies

```bash
pip install --upgrade pip
pip install -r requirements.txt
```

This installs:

- **mlflow** - Experiment tracking and model management
- **scikit-learn** - Machine learning algorithms
- **tensorflow & keras** - Deep learning library (may take 2-3 minutes)
- **pandas & numpy** - Data manipulation
- **jupyter & ipykernel** - Notebook environment
- **hyperopt** - Smart hyperparameter tuning

### 4. (Optional) Setup Jupyter Kernel

This ensures Jupyter uses the correct Python environment:

```bash
python -m ipykernel install --user --name=mlflow_starter --display-name "Python (MLFlow Starter)"
```

## 🏃‍♂️ Running the Project

This project requires TWO terminal windows running simultaneously!

### Step 1: Start the MLflow Tracking Server (Terminal 1)

Open the FIRST terminal:

```bash
# Make sure your virtual environment is activated
# (.venv) should appear at the start of your terminal line

# Start the MLflow tracking server
mlflow server --host 127.0.0.1 --port 5000
```

Output will look like:

```
[2026-03-19 10:00:00 +0000] [12345] [INFO] Starting gunicorn 20.1.0
[2026-03-19 10:00:00 +0000] [12345] [INFO] Listening at: http://127.0.0.1:5000
```

✅ **KEEP THIS TERMINAL OPEN** while working with notebooks!

👉 **View MLflow Dashboard**: Open `http://127.0.0.1:5000` in your web browser to see all experiments and runs

### Step 2: Launch Jupyter Notebook (Terminal 2)

Open a SECOND terminal (keep the first one running):

```bash
# Activate the virtual environment in this terminal too
# On Windows: .\.venv\Scripts\Activate.ps1
# On Mac/Linux: source .venv/bin/activate

# Start Jupyter
jupyter notebook
```

Jupyter will automatically open in your browser at `http://localhost:8888`

### Step 3: Run the Notebooks!

**IMPORTANT:** Before running cells, select the correct kernel:

1. Click the kernel selector in the top right corner (shows "Select Kernel")
2. Choose `Python (MLFlow Starter)` (if you created it in setup step 4)
3. Or select your venv Python kernel

**Recommended Learning Path:**

1. `get-started.ipynb` (5-10 min) - MLflow basics
2. `1-MLproject/gettingstarted.ipynb` (15-20 min) - Train real model
3. `1-MLproject/housepricepredict.ipynb` (30-45 min) - Hyperparameter tuning
4. `2-DLFLOW/starter.ipynb` (45-60 min) - Deep learning + optimization

**Troubleshooting:**

- **"Connection refused"**: Is Terminal 1 (MLflow server) still running?
- **Slow to start**: TensorFlow is loading (normal, 30-60 seconds first run)
- **Kernel won't start**: Run `pip install --force-reinstall ipykernel`
